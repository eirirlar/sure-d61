  D6.2 — Topic, scope, and readiness                                                                
                                                                                                    
  What D6.2 should be about

  D6.1 delivered the framework specification: a conceptual model chain, domain-specific models in
  early form, and partially manual data flows — with explicit deferral of full integration to D6.2.

  D6.2 should be the working implementation: the same model chain now actually running end-to-end,
  applied to make concrete design decisions on the next-gen floater. The grant description for Task
  6.2 is explicit — it wants:

  ▎ "Digital product development cycle implementation, using all key design parameters... Especially
  ▎  important is harmonizing input and output data from the different models to enable automated
  ▎ mapping of the parameter space and rapid iteration turnover."

  In short: D6.1 = specifications and groundwork. D6.2 = the thing actually working and producing
  optimisation results.

   load version or document the interface design even if not yet live.

  ---
  Things in D6.1 that might be premature for D6.1 and more naturally live in D6.2

  Nothing should be moved at this point since D6.1 is being delivered — but the following are
  intentionally thin in D6.1 and should grow substantially in D6.2:

  - Structural modelling results — the current SiSim simulation had no infill, making it a partial
  picture. D6.2 should have the full model with infill, and ideally validated against P4/P5
  prototype measurements.
  - White PU thermal properties — absorptivity was measured for dark blue PU. Off-white values are
  needed and missing. D6.2 should have these and the updated thermal simulation.
  - UV/adhesion testing conclusions — currently "in progress" for P3. D6.2 should report outcomes
  and what design rules they produced.
  - Screening across domains — D6.1's Chapter 7 is only manufacturing feasibility screening. D6.2
  should show what happens when you also apply structural, thermal and LCA filters — i.e., what
  fraction of the ~1,300 feasible formed geometries survive all constraints.

  ---
  Work that should happen before delivering D6.2

  Roughly in priority order:

  High priority (blocking the core D6.2 claim):
  1. Resolve the [TODO for IFE] items in interfaces I-2, I-3, I-4 — agree on STEP version, PATRAN
  format for thickness map, material property handover protocol. Even if automation is not complete,
   the specs must be formal.
  2. Agree the LCA data template with TNO and ensure it's linked to the FreeCAD mass model, not to
  ad hoc email tables.
  3. Measure absorptivity of off-white PU and run updated thermal CFD for P4 geometry (this was
  explicitly flagged as "plan videre" in the WP6 report).
  4. Complete the 3DFloat interface design (even if only preliminary loads are available) so D6.2
  can demonstrate at least one end-to-end run from wave conditions to structural stresses.

  Important for design content:
  5. Complete P4 casting and run the reusable mechanical simulation rig (load cases 1a–1f,
  measurement points 2a–2f) that was set up as infrastructure in D6.1.
  6. Complete white PU UV and adhesion testing (midpatch samples were at IFE awaiting test planning
  as of the T6.2 report).
  7. Get TNO's LCA results (expected Q2 2026) so D6.2 can show which design variables matter most
  for CO₂ footprint.

  Longer horizon (for P5/P6 convergence):
  8. Resolve the architecture question (cast-on-frame vs separate-and-mount) — this is the key
  decision that unlocks the final floater design, and D6.2 should be able to say which way it went
  and why.
  9. Apply the multi-domain screening (manufacturing + structural + thermal + LCA) to a concrete
  shortlist of design candidates and demonstrate that the model chain guided the selection — this is
   the headline result that justifies D6.2's existence.

  ---
  Summary framing

  D6.2's core claim should be: "We built the model chain described in D6.1, connected the tools, ran
   it on the evolving prototype design, and here are the design decisions it enabled." The concrete
  deliverable is the implemented interfaces table from D6.1 with all "manual / targeted for D6.2"
  entries upgraded to "implemented / automated."

  The single biggest risk is the 3DFloat coupling — if that's not available until late 2026, D6.2
  should clearly scope whether it delivers the interface specification only, or a first live run
  with preliminary wave loads.
